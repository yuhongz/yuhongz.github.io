url到页面加载过程

  • dns解析
  • 浏览器根据ip地址向服务器发送请求：三次握手建立tcp连接
  • 服务器处理http请求，并返回给浏览器
  • 渲染过程1
    • 根据html生成dom树
    • 根据css生成cssom
    • dom tree和cssom整合形成render tree
  • 渲染过程2
    • 根据render tree渲染页面
    • 遇到script暂停渲染，优先加载并执行js代码，完成再继续
    • 直至把render tree渲染完成

window.onload和DOMContentLoaded

  • 渲染遇到img标签不会阻塞，而是会继续向后。img加载完成后可能触发重绘。
window.addEventListener('load', function(){
    // 页面的全部资源加载完 包括图片 视频
})
document.addEventListener('DOMContentLoaded'， function(){
    // DOM渲染完即可执行，此时图片和视频可能还没加载完。很多情况使用此事件更合理
})

🌟性能优化！🌟

  • 原则
    • 多使用内存、缓存
    • 减少cpu计算，减少网络加载耗时
  • 做法
    • 让加载更快
      • 减少资源体积：压缩代码
      • 减少访问此时：合并代码，ss服务器端渲染，缓存
      • 更快的网络：使用cdn
    • 让渲染更快
      • css放在head，js放在body的下部
      • 用domcontentloaded触发而不是window.onload，更快触发js
      • 懒加载
      • DOM查询缓存
      • 合并DOM操作
      • throttle 和 debounce

//防抖：停止后一段时间执行
function debounce(fn, delay = 500){
    //timer是闭包中的
    let timer;
    return function(){
        if(timer) clearTimeout(timer);
        timer =  setTimeout(()=>{
            fn.apply(this, arguments);//apply 不是 call
            timer = null; //必须，timer=null与cleartimeout(timer)不一致！！后者timer仍然有值
        }.delay)
    }
}

//截流：无论触发多频繁，都是固定时间段执行一次/也会延迟！
function throttle(fn, delay = 500){
    let timer;
    return function(){
        if(timer) return;
        timer =  setTimeout(()=>{
            fn.apply(this, arguments);//apply 不是 call
            timer = null; //必须，timer=null与cleartimeout(timer)不一致！！后者timer仍然有值
        }.delay)
    }
}

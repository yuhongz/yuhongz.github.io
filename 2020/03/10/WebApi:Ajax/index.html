🌟XMLHttpRequest

//手写xhr
const xhr = new XMLHttpRequest();
xhr.open('GET', 'data.json', true) //异步用true，可支持同步false。
xhr.onreadystatechange = function(){ //无参数
    if(xhr.readyState === 4){
        if(xhr.status === 200){
            alert(JSON.parse(xhr.responseText)) //responseText!  json解析字符串
        }
    }
}
xhr.send(null); //发送数据

//如post需要发送数据
const postData = {};
xhr.send(JSON.stringify(postData)); //转字符串

  • xhr.readyState：0：未调用send；1:已调用send；2:接收到全部响应；3解析响应内容；4完成解析，可在客户端调用。

  • Xhr.state:

    • 2xx：表示成功
    • 3xx：重定向，浏览器直接跳转。301永久重定向；302临时重定向；304读取缓存。
    • 4xx：客户端请求错误，404没有资源，403没有权限
    • 5xx：服务端出错

    403缓存：在http响应头中设定，相关字段：expires, cache-control, E-tag, LastModified。cache-control优先级大于expires，e-tag大于lastmodified。cachecontrol中的maxage设置了时长，单位秒

🌟 同源策略

  • ajax请求时，浏览器要求当前页面和sever必须同源。（安全）
  • 同源：协议、域名、端口一致。
  • 无视同源策略的几种情况
    • 图片<img src=''/>
    • js<script src=''></script> (JSONP的基础)
    • css<link href=''/>

🌟跨域

  • 所有的跨域都要与服务端进行配合

  • JSONP：

    • script可以绕过同源策略

    • 服务端可以拼接动态数据返回

  • CORS：服务器设置http header

    response.setHeader('Access-Control-Allow-Origin')...

promise实现ajax

function ajax(url){
    return new Promise((resolve, reject) => {
        const xhr = new XMLHttpRequest();
        xhr.open('GET', url, true);
        xhr.onreadystatechange = function(){
            if(xhr.readyState === 4){
                if(xhr.status === 200){
                    resolve(JSON.parse(xhr.responseText))
                }
                else reject(new Error('error'))
            }
        }
        xhr.send(null)
    })
}
ajax(url).then((res)=>{console.log(res)}).catch((err) => {console.error(err)});

Fetch

fetch(url); //返回一个promise
  • 与jquery.ajax区别：
    • 当状态码为4xx和5xx时，也会resolve
    • fetch不会从服务端发送或接收cookie

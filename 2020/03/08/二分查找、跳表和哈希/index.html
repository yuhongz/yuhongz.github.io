1.二分查找

  • 优点：

    • o(logn)复杂度的查找效率，非常恐怖
    • 节省内存，同样的查找工作虽然哈希和跳表也可以做，但他们需要额外空间

  • 限制：

    • 有序
    • 只能基于顺序表，即数组，因为要实现随机访问。
    • 数据量太大或太小都不合适，太小不如直接遍历，数据量大需要很多连续空间。

  • 代码

    • 非递归，不存在重复元素

      function binSearch(A, n, k){
    var low = 0;
    var high = n-1;
    var mid;
    while(low <= high){
        mid = Math.floor((high + low)/2);
        if(A[mid] === k) return mid;
        if(A[mid] > k) high = mid - 1;
        if(A[mid] < k) low = mid + 1;
    }
    return -1;
}

    • 以上代码三个要点：

      • 循环条件 low <= high 不是<
      • high和low的更新，high=mid-1而不是mid，否则会死循环
      • mid = (high + low) / 2 可能会溢出，可以写成low + (high-low)/2

    • 递归实现

      function binSearch(A, n, k){
    binSearch_c(A, 0, n-1, k);
}
function binSearch_c(A, low, high, k){
    if(low > high) return -1;
    var mid = Math.floor((high+low)/2);
    if(A[mid] === k) return mid;
    if(A[mid] > k) return binSearch_c(A, low, mid-1, k); //需要返回
    if(A[mid] < k) return binSearch_c(A, mid+1, high, k);
}
